# 焦点检测

## 需求

- 检测目标游戏窗口是否在前台
- 焦点丢失时自动暂停回放，恢复后继续
- 提供可见窗口列表供用户选择

## 设计方案

### FocusWatcher.py

纯函数模块，通过 ctypes 调用 Win32 API：

| 函数 | Win32 API | 用途 |
|------|-----------|------|
| `GetForegroundWindowTitle()` | `GetForegroundWindow` + `GetWindowTextW` | 获取当前前台窗口标题 |
| `IsWindowFocused(Title)` | 同上 | 判断标题是否包含在前台窗口标题中（大小写不敏感） |
| `ListWindows()` | `EnumWindows` + `IsWindowVisible` + `GetWindowTextW` | 枚举所有可见窗口标题 |

### 焦点匹配策略

使用 `in` 子串匹配而非精确匹配（`Title.lower() in CurTitle.lower()`），因为部分游戏窗口标题会带额外信息。Title 为空时视为匹配（不做焦点检测）。

### 与回放引擎的集成

KeyPlayer 在每次发送按键前调用 `IsWindowFocused`：
- 匹配 → 正常发送
- 不匹配 → 进入暂停轮询（200ms 间隔），emit `StatusChanged("paused")`
- 恢复匹配 → emit `StatusChanged("playing")`，继续发送

## 边界问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 窗口标题动态变化 | 子串匹配容忍标题变化，用户输入核心关键词即可 |
| 目标窗口未设置 | Title 为空时 IsWindowFocused 直接返回 True，跳过检测 |
| EnumWindows 回调中异常 | 回调用 WINFUNCTYPE 装饰，内部无异常抛出 |
