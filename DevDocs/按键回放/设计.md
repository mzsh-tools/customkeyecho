# 按键回放

## 需求

- 用户录制一组键盘/鼠标按键序列
- 按快捷键启动循环回放，再按停止（toggle 模式）
- 每次按键间隔从用户设定的 [Min, Max] 区间随机取值
- 一轮完成后自动从头循环
- 间隔下限 10ms

## 设计方案

### 数据模型（KeyProfile.py）

```python
@dataclass
class KeyAction:
    Type: str  # "Key" / "Mouse"
    Code: int  # 键盘=VK 码，鼠标=按键编号(1~5)
    Name: str  # 显示名称
```

### 录制流程（InputListener.py）

单一 pynput 键盘监听 + 模式切换，避免多监听器冲突：
- `idle`：监听快捷键触发
- `recording`：捕获所有按键添加到序列，回车结束，同时启动鼠标监听
- `setting_hotkey`：捕获下一个按键作为快捷键

录制时额外启动 pynput 鼠标监听（仅捕获按键，不记录坐标），录制结束时停止。

### 回放引擎（KeyPlayer.py）

QThread 子类，通过 threading.Event 控制停止：

```
while not StopEvent:
    for Action in Actions:
        检测焦点 → 发送按键 → random(Min, Max) 延迟
    自动进入下一轮
```

- 焦点丢失时进入暂停轮询（200ms 间隔），恢复后继续
- StopEvent.wait(Delay) 同时实现延迟和快速响应停止
- StatusChanged 信号通知 UI 状态变化（playing/paused/stopped）

### 快捷键 toggle（MainWindow._TogglePlayback）

```python
if Player.isRunning():
    Player.Stop()
elif Actions 非空:
    Player.Configure(...) + Player.start()
```

用 isRunning() 而非额外标志位，正确处理 playing/paused/stopped 所有状态。

## 边界问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 回车键无法录入序列 | 回车用于结束录制，暂不支持；后续可增加 UI 手动添加功能 |
| 录制时鼠标点击 UI 被捕获 | 录制按钮的 click 信号在鼠标释放后触发，pynput 监听在此之后启动，首次点击不会被误录 |
| DelayMin > DelayMax | UI 层 SpinBox 联动校验，修改 Min 时自动调整 Max，反之亦然 |
| 回放中修改设置不生效 | Configure() 在 start() 前调用且拷贝列表，运行中设置变更需停止后重启 |
| 快捷键快速连按 | isRunning() 防止重复启动；Stop() 幂等，重复调用无副作用 |

## 废弃方案

| 方案 | 废弃原因 |
|------|----------|
| 多个 pynput Listener 分别监听 | 多监听器可能冲突，单监听 + 模式切换更可靠 |
| _IsPlaying 标志位控制 toggle | 焦点暂停时标志位与 isRunning() 不一致，直接用 isRunning() 更准确 |
